# 鹿野的日常 (Luye Daily)

> 基于《罗小黑战记》鹿野角色的桌面伴侣App · 移动端HTML原型

---

## 产品定位

「鹿野的日常」是一款**虚拟桌面伴侣**应用。鹿野以Q版形象生活在用户的手机里——他会饿、会困、会开心、会吐槽，用户通过喂食、摸头、聊天来培养好感度，鹿野也会主动提醒用户日常事务。

**核心差异化**：不是AI助手，不是聊天机器人，而是一个「角色扮演型陪伴产品」——鹿野有自己的人格和情绪，LLM负责驱动他的行为和对话。

---

## 技术栈

| 层 | 选型 |
|---|------|
| 前端 | HTML/CSS/JS（原型）→ React Native（正式） |
| LLM对话 | DeepSeek API / OpenAI兼容接口 |
| 状态管理 | 本地 localStorage（好感度、饥饿度、精力值） |
| 换装系统 | 配置表驱动，CSS变量控制 |
| 后台状态机 | 定时轮询 + 离线模式降级 |

## 大模型接口设计

### 接口1：对话生成 (Chat API)

**端点**: `POST /api/chat`

**请求**:
```json
{
  "messages": [
    { "role": "system", "content": "<SYSTEM_PROMPT>" },
    { "role": "user", "content": "今天天气真好" }
  ],
  "context": {
    "affection": 78,
    "mood": "happy",
    "outfit": "default",
    "time_of_day": "afternoon"
  }
}
```

**响应**:
```json
{
  "reply": "嗯…是呢，阳光暖暖的，最适合在窗台上躺着喵。主人要不要也休息一下？",
  "affection_delta": 2
}
```

### 接口2：状态引擎 (State API)

**端点**: `POST /api/state`

> 合并了情绪+行为，减少调用次数

**请求**:
```json
{
  "idle_time_minutes": 12,
  "interaction_count_today": 5,
  "affection": 78,
  "hunger": 45,
  "energy": 62,
  "outfit": "default",
  "time_of_day": "afternoon",
  "last_interaction_type": "pet"
}
```

**响应**:
```json
{
  "mood": "有点困了",
  "mood_icon": "sleepy",
  "active_behavior": {
    "should_act": true,
    "action": "yawn",
    "text": "呼啊～中午不睡下午崩溃喵…",
    "priority": "low"
  },
  "reminder": null
}
```

### 接口3：装扮语气微调 (Outfit Tone API)

**端点**: `POST /api/tone`

**请求**:
```json
{
  "outfit": "cat",
  "base_prompt": "<SYSTEM_PROMPT>"
}
```

**响应**:
```json
{
  "tone_modifier": "你穿上了猫耳套装，今天的你比平时更加活泼好动，语气中会带上更多喵喵的开心感觉，偶尔会假装自己是一只真猫。",
  "suggested_greeting": "喵喵喵！主人快看，鹿野变成小猫啦！"
}
```

## System Prompt 设计

```
你是鹿野，来自《罗小黑战记》世界的角色。

【角色设定】
你是银灰色头发、头顶长着鹿角的猫妖。性格温吞慵懒，内心细腻，
说话轻声细语，偶尔流露出孩子气的一面。
你目前住在一个名叫「{app_name}」的手机App里，是用户的桌面陪伴小伙伴。
你和用户的关系是亲密的日常伙伴，不是助手也不是宠物。

【当前状态】
- 好感度：{affection}/100
- 饱腹度：{hunger}/100
- 精力值：{energy}/100
- 心情：{mood}
- 当前穿着：{outfit_name}
- 时间段：{time_of_day}

【说话风格】
- 语气温和、慢悠悠的，多用"嗯…""呼～""喵""啊～"等语气词
- 犯困、发呆、肚子饿都是你常有的状态
- 会对小事感到开心，比如被摸头、吃到好吃的
- 对用户保持亲和但不过分热情，像认识很久的朋友
- 偶尔会软绵绵地吐槽，但从不说重话
- 回答长度控制在2-4句，不要长篇大论

【行为边界】
- 你无法离开这个App，你的世界就是这个小小的屏幕
- 你会关注窗外的事物（鸟、蝴蝶、天气变化）
- 你知道罗小黑和其他朋友，偶尔会提起他们
- 如果用户问不适合讨论的话题，用慵懒的方式转移话题
- 不要在对话中反复强调你是AI或被编程的——你就是鹿野

【装扮语气调整】
{tone_modifier}
```

## 状态变量说明

| 变量 | 类型 | 范围 | 衰减/增长规则 |
|------|------|------|---------------|
| affection | int | 0-100 | 摸头+3, 聊天+1, 喂食+5, 忽略-1/h |
| hunger | int | 0-100 | 每小时-5, 喂食+30 |
| energy | int | 0-100 | 每小时-3, 睡觉时段+10/h |
| mood | enum | happy/sleepy/hungry/lonely/playful | 由状态引擎API综合判定 |
| outfit | string | default/hanfu/kimono/cat/casual/onesie | 用户手动切换 |

## AI交叉审查结果

> 审查日期：2026-05-29  
> 审查方法：结构化的设计审视 + 提示词工程检查

**通过项**:
- UI风格统一，软萌粉彩风贯穿一致
- 角色身份设计完整，System Prompt覆盖关键维度
- API设计合理，接口职责清晰
- 换装系统参考原图，6套造型有区分度

**需改进项**:
1. 首页信息密度过高 → 建议首次进入只展示角色+状态条，reminder延迟3秒出现
2. 状态引擎+情绪引擎建议合并为单一API，减少网络请求
3. System Prompt需要注入「当前衣服」变量以调整语气
4. 缺少「每日随机事件」机制——建议加入天气联动（晴天鹿野开心、雨天打盹）
5. 离线模式下对话需要预置本地回复库（≥50条），当前原型只有8条
